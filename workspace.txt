SETTING UP WORKSPACE IN PyCharm

1. SETUP GITHUB
mkdir structural-gt
cd structural-gt/
git init
git remote add origin https://github.com/owuordickson/structural-gt.git
git fetch origin
git fetch origin cppy
git pull origin cppy

2. INSTALL gcc/g++ (C/C++ COMPILER)
Linux/MacOS:
use 'brew install gcc'
Windows:
download and install 'Microsoft C++ Build Tools' (install Desktop development with C++)
* Use 'Launch' and command where.exe to find: clang/cl and cmake and nmake
--
download mingw-get-setup.exe installer
run setup As Administrator
install only 'mingw32-gcc-g++' and 'mingw32-make' and 'mingw32-pthreads-w32'
Add path 'C:\MinGW\bin' and 'C:\MinGW\msys\1.0\bin' to Environment Variables
test 'gcc --version'

3. INSTALL CMAKE
Linux/MacOS: use 'brew install cmake' 
Windows:
use 'Microsoft C++ Build Tools' to install
--
download and install CMAKE from https://cmake.org/
Add CMAKE path to Environment Variables
test 'cmake --version'

4. C LIBRARIES
4a . PTHREADS library
Linux/macOS:
Already installed
Windows:
Install Microsoft Build Tools
We create the library files using 'nmake all install' through 'Developer Command Prompt'
Copy header files and .dll files to MinGW folder

4b. IGRAPH library
Linux/macOS:
brew install igraph
Windows:
Install Microsoft Build Tools
https://igraph.org/c/html/latest/igraph-Installation.html#igraph-Installation-installation
Use the 'Developer Command Prompt' to run the following commands:
mkdir build
cd build
cmake ..
cmake --build . --config Release
cmake --build . --target check --config Release
Copy (include) header files and igraph.lib files to MinGW folder

5. COMPILING in WINDOWS
clang .\test.c -o .\test.exe
gcc .\test.c -o .\test

Windows:
gcc .\igraph_test.c -I../c_modules -IC:/MinGW/include/igraph -IC:/MinGW/include/pthread -LC:/MinGW/lib/igraph -LC:/MinGW/lib/igraph -ligraph -lpthread -o .\igraph_test
gcc .\sgt_test.c -I../c_modules -IC:/MinGW/include/igraph -IC:/MinGW/include/pthread -LC:/MinGW/lib/igraph -LC:/MinGW/lib/igraph -ligraph -lpthread -o .\sgt_test

6. SETUP PYCHARM or VS CODE
